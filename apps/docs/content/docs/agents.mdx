`@ziro-agent/agent` ships a minimal but production-ready ReAct-style loop. It
calls the model, executes any tools it requests, threads the results back
into the conversation, and repeats until a stop condition is met.

## Quick start

```ts
import { createAgent } from '@ziro-agent/agent';
import { defineTool } from '@ziro-agent/tools';
import { z } from 'zod';

const search = defineTool({
  name: 'search',
  description: 'Search the knowledge base.',
  input: z.object({ q: z.string() }),
  execute: async ({ q }) => ({ hits: [`result for ${q}`] }),
});

const agent = createAgent({
  model,
  tools: { search },
  maxSteps: 5,
});

const result = await agent.run({
  prompt: 'Find docs about streaming.',
  onEvent: (event) => console.log(event.type),
});
```

## Stop conditions

Combine the helpers from `@ziro-agent/agent` to terminate early:

```ts
import { createAgent, anyOf, stepCountIs, totalTokensExceeds } from '@ziro-agent/agent';

createAgent({
  model,
  tools,
  stopWhen: anyOf(stepCountIs(8), totalTokensExceeds(20_000)),
});
```

## Events

Each `AgentRunOptions.onEvent` callback receives a discriminated union:
`step-start`, `llm-finish`, `tool-result`, `step-finish`, `agent-finish`. Use
this for live UI updates or shipping to your tracing backend.

## Tool tracing

Set **`traceTools: true`** on `createAgent` to wrap every tool (including
handoff tools) with `instrumentTools` from `@ziro-agent/tracing`. Install a
tracer with `setTracer(...)`; until then spans are no-ops. Leave `traceTools`
off (default) if you already pass a pre-wrapped tool map to avoid double spans.

See [Tracing](/docs/tracing) and [Sandbox & browser tools](/docs/sandbox-browser).

## Durable execution (Inngest)

To survive process restarts and avoid re-billing the same LLM step on retry,
wrap the agent with **`createInngestAgent`** from `@ziro-agent/inngest` and a
`Checkpointer`. Walkthrough: [Durable Inngest cookbook](/docs/cookbooks/durable-inngest).
